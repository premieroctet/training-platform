#### Un composant React

React permet de définir des composants à lesquels on passe des `props` :

```jsx
// Avatar.jsx

const Avatar = ({ avatarSrc, username }) => (
  <div>
    <img src={avatarSrc} alt={username} width="60" height="60" />
    {username}
  </div>
);
```

#### Le markup de React : JSX

- Un composant ne **retourne pas directement** du HTML mais du **JSX**
- C'est une extension de la syntaxe JS
- Permet d'écrire des syntaxes plus complexes que du HTML (conditions,
  boucles...)
- Permet à React d'appliquer son algorithme de **réconciliation** ⚡️

#### Composant en TypeScript

On peut utiliser TypeScript pour rendre **plus robuste** ses composants, pour cela il faut :

- Utiliser l'extension `.tsx`
- Déclarer les types de nos props

---

<SlideHeader slideTitle="TypeScript et React" />

#### Composant en TypeScript

Exemple de composant avec des props typés :

```tsx
// Avatar.tsx

type AvatarProps = { avatarSrc: string; username: string };

const Avatar = ({ avatarSrc, username }: AvatarProps) => (
  <div>
    <img src={avatarSrc} alt={username} width="60" height="60" />
    {username}
  </div>
);
```

#### JSX : en coulisse

Voici comment est transpilé du JSX :

```jsx
const element = <h1 className="greeting">Hello, world!</h1>;

const element = React.createElement(
  "h1",
  { className: "greeting" },
  "Hello, world!"
);
```

On peut configurer comment est transpilé le JSX [dans le tsconfig](https://www.typescriptlang.org/docs/handbook/jsx.html)
