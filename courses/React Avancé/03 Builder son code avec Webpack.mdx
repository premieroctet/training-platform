##### <b>Le bundling :</b> qu'est ce que c'est ? 🤔

- La multiplication du nombre de fichiers JavaScript dans une page web peut
  devenir un frein aux performances
- Un bundler est un outil qui regroupe du code et toutes ses dépendences en un
  seul fichier.
- Le temps de chargement est réduit en même temps que le nombre de requêtes pour
  des fichiers JavaScript.

##### Utiliser <b>Webpack</b> comme outil de build 🚀

- https://webpack.js.org/
- Organise l'application JS en bundles, mais également les autres ressources
  nécessaires (css, images…)
- Offre un serveur de développement permettant le hot reload 🔥

---

<SlideHeader slideTitle="Builder son code avec Webpack" />

#### 📦 Créer un simple bundle

- On ajoute Webpack localement en dépendances (dev)
- On peut alors créer un bundle de notre code (src/ vers build/)

```
yarn add --dev webpack
./node_modules/.bin/webpack src/index.js
```

---

<SlideHeader slideTitle="Builder son code avec Webpack" />

#### 📦 Créer un simple bundle

- Comme pour Babel, on utilise le champ "**scripts**" du fichier
  **package.json** 😎

```json
{
  "scripts": {
    "webpack": "webpack src/index.js"
  },
  "devDependencies": {
    "webpack": "^4.5.1",
    "webpack-cli": "^3.1.2"
  }
}
```

##### Gestion de la **config Webpack**

- Nous avons utilisé Webpack avec la config par défaut
- Nous pouvons personnaliser la phase de build, en ajoutant le fichier
  **webpack.config.js**
- Nous commencons par définir **le point d'entrée** et **le point de sortie** :

```js
var path = require("path");

module.exports = {
  entry: "./src/index.js",
  output: {
    filename: "bundle.js",
    path: path.resolve(__dirname, "build"),
  },
};
```

##### Les loaders **Webpack** 🚩

- https://webpack.js.org/concepts/loaders/
- Les loaders déterminent quels traitements appliquer aux différentes ressources
  gérées par Webpack
- Il existe une multitude de loaders :
- [babel-loader (pour transpiler le js)](https://webpack.js.org/loaders/babel-loader/)
- [css-loader (pour build les css)](https://webpack.js.org/loaders/css-loader/)
- [file-loader (pour build les assets)](https://webpack.js.org/loaders/file-loader/)

---

<SlideHeader slideTitle="Builder son code avec Webpack" />

##### Les loaders **Webpack** 🚩

- On les renseigne dans le tableau **rules** de notre fichier de configuration :

```js
var path = require("path");

module.exports = {
  entry: "./src/index.js",
  output: {
    filename: "bundle.js",
    path: path.resolve(__dirname, "build"),
  },
  module: {
    rules: [{ test: /\.css$/, use: "css-loader" }],
  },
};
```

##### Les plugins **Webpack** ✨

- Les **plugins** apportent de nouvelles fonctionnalités à Webpack comme -
  [webpack-bundle-analyzer](https://github.com/webpack-contrib/webpack-bundle-analyzer)
  qui permet de visualiser ses dépendances :

```js
var path = require("path");
const BundleAnalyzerPlugin =
  require("webpack-bundle-analyzer").BundleAnalyzerPlugin;

module.exports = {
  entry: "./src/index.js",
  output: {
    filename: "bundle.js",
    path: path.resolve(__dirname, "build"),
  },
  plugins: [new BundleAnalyzerPlugin()],
};
```

##### Les modes **Webpack** 🚀

- Il existe deux modes: `production` et `developpement`

```js
// Via la config
module.exports = {
	// ...
	mode: 'production'
};

// Via le cli
webpack --mode=developpement
```
