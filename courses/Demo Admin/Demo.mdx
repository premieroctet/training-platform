# [MDX Provider](https://mdxjs.com/getting-started/#mdxprovider)

- MDX Provider takes an `component` object that overrides any markdown/html &
  react tags you want to customize.

```jsx
// src/components/mdx/MDXProvider.tsx

import React from "react";
import { MDXProvider } from "@mdx-js/react";
// import my components ...
const mdComponents = {
  h1: (props: any) => <Heading fontSize="2.4em" {...props} />,
  h2: (props: any) => <Heading fontSize="2em" {...props} />,
  a: (props: any) => <a target="_blank" {...props}></a>,
  code: (props: any) => <CodeBlock {...props} />,
  ChapterHeading,
  SlideHeader,
  // ...
};

export default ({ children }) => (
  <MDXProvider components={components}>{children}</MDXProvider>
);
```

#### Built-in components

- the `Slides` component wraps up your mdx to display it as slides : only write
  mdx slides divided with `---` OR `<hr />`

```jsx
<Slides title="chapter title" course="course title">
  // your mdx slides
</Slides>
```

#### use components to easily customize layout

the content of each slide is wrapped in a basic flex column container, but you
can import and use any component or html tag to build your layout.

[ChakraUI](https://chakra-ui.com/) provides very usefull
[layout components](https://chakra-ui.com/docs/layout/stack) to create Verical &
